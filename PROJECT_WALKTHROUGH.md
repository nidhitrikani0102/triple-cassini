2.  **Install Dependencies**:
    *   **Root (Concurrently)**:
        ```bash
        npm install
        ```
    *   **Server**:
        ```bash
        cd server
        npm install
        ```
    *   **Client**:
        ```bash
        cd ../client
        npm install
        ```

### Environment Setup
Create a `.env` file in the `server/` directory with the following credentials:
```env
PORT=5000
MONGO_URI=mongodb://localhost:27017/eventempire
JWT_SECRET=your_jwt_secret_key_here
EMAIL_SERVICE=gmail
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_app_password
CLIENT_URL=http://localhost:3000
```

### Running the Application
From the root directory, run:
```bash
npm run dev
```
*   **Client**: Runs on `http://localhost:3000`
*   **Server**: Runs on `http://localhost:5000`

## ðŸ”„ 4. Key Workflows

### Vendor Assignment Lifecycle
1.  **User** sends request -> Status: **Pending**.
2.  **Vendor** accepts -> Status: **In Progress**.
3.  **Vendor** completes work -> Status: **Completed**.
4.  **User** pays -> Status: **Paid** (Expense added to Budget).

### Invitation System
1.  **User** adds Guest (Email or In-App).
2.  **System** sends email with RSVP link.
3.  **Guest** clicks link -> Updates status to **Accepted/Declined**.
4.  **User** sees real-time status updates on Event Page.

## ðŸ”§ 5. Troubleshooting

*   **MongoDB Connection Error**: Ensure MongoDB is running locally (`mongod`) or your Atlas URI is correct in `.env`.
*   **Email Not Sending**: Check `EMAIL_USER` and `EMAIL_PASS`. For Gmail, use an "App Password", not your login password. Check server console for fallback logs if email fails.
*   **Images Not Loading**: Ensure the `uploads/` folder exists in `server/` and you are running the server.
